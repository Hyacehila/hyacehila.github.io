## 0. 文档定位

本文档用于同时服务三类目标：

1. **人类学习**：系统理解小红书起号、养号、内容生产和接广告的通用方法。
2. **AI 检索/训练**：将经验拆成结构化概念、字段、流程、规则和模板，便于后续作为知识库调用。
3. **自动化机器人设计**：为后续开发合规的起号/养号辅助机器人提供需求边界、模块划分和输入输出格式。

本文档讨论的“自动化机器人”不是灰产工具，不包含批量注册、批量发布、刷赞刷藏、机器评论、伪造互动、绕过风控等能力。机器人只负责内容策略、文案生成、封面建议、发布计划、数据复盘、商单资料管理和风险提醒；真实发布、互动、商业承诺与账号操作必须由人工确认。

参考资料：

- 小红书社区规范：禁止批量注册、机器/高频发布、程序或脚本制造虚假互动数据等作弊行为。
- 小红书蒲公英平台：包含内容合作、订单管理、收入明细、合作设置、报价设置、标签管理、数据详情和粉丝画像等能力，可作为商单管理模块的参考。

## 1. 平台认知

### 1.1 小红书的核心内容逻辑

小红书不是单纯的信息流平台，也不是单纯的搜索平台，而是“信息流发现 + 搜索决策 + 种草信任”的混合场。

| 平台机制 | 对创作者的含义 | 对机器人的要求 |
| --- | --- | --- |
| 信息流发现 | 标题、封面、前几行决定是否被点击 | 生成多套标题/封面钩子并做 A/B 记录 |
| 搜索心智 | 用户带着明确问题搜索攻略、测评、经验 | 选题需要沉淀关键词、场景词、问题词 |
| 种草信任 | 用户相信真实体验、细节和可验证信息 | 文案必须避免虚假体验和夸大承诺 |
| 收藏行为 | 实用内容、清单内容、教程内容更容易被收藏 | 生成“可保存”的结构化内容 |
| 评论互动 | 观点、选择题、补充经验能带来讨论 | 评论引导应自然、合规、非诱导作弊 |

### 1.2 起号的底层公式

起号不是“发很多笔记”这么简单，而是持续让平台和用户理解账号是谁、给谁看、解决什么问题。

```text
账号增长 = 清晰定位 × 稳定内容 × 可识别风格 × 有效互动 × 持续复盘
```

关键判断：

- **定位不清**：用户不知道为什么关注，平台也难以识别账号标签。
- **内容不稳**：今天美妆、明天职场、后天旅游，会稀释推荐信号。
- **价值不明**：内容只有观点，没有清单、方法、案例、对比、避坑点，收藏动力弱。
- **表达不平台化**：内容有价值但标题、封面、开头太像报告，用户没有耐心进入正文。
- **缺少复盘**：只凭感觉发文，无法知道哪类选题、标题、封面和结构有效。

### 1.3 “先吸引，再转化”

账号早期的主要任务不是立即卖货或接广告，而是建立账号的可消费理由。

推荐节奏：

1. **先建立内容消费理由**：让用户知道账号能持续提供什么价值。
2. **再建立信任资产**：通过真实体验、稳定输出、案例细节、评论互动积累可信度。
3. **最后承接商业转化**：通过商单、私信、蒲公英合作或品牌长期合作变现。

可用内容结构：

- 60%：垂直领域的专业/实用内容，用于建立账号价值。
- 30%：高传播潜力内容，用于扩大曝光和吸粉。
- 10%：商业转化或合作内容，用于测试变现能力。

比例不是固定规则，机器人应允许按账号阶段调整。

## 2. 起号模型

### 2.1 账号定位

账号定位需要回答 6 个问题：

| 字段 | 说明 | 示例 |
| --- | --- | --- |
| 领域 | 账号长期讨论的主题 | 职场效率、护肤测评、低成本旅行 |
| 人群 | 内容主要服务谁 | 新手妈妈、应届生、敏感肌用户 |
| 场景 | 用户在什么情况下需要内容 | 入职前、换季护肤、周末出行 |
| 痛点 | 用户当前有什么问题 | 不会选、不敢买、时间少、预算低 |
| 价值 | 账号提供什么确定收益 | 省钱、省时间、避坑、提升效率 |
| 边界 | 哪些内容不做 | 不接医疗疗效、不做虚假体验、不引战 |

推荐定位句式：

```text
为【目标人群】提供【领域/场景】中的【核心价值】，重点解决【高频痛点】，不做【内容边界】。
```

### 2.2 账号标签

账号标签不是单个标签，而是一组长期稳定的信号。

| 标签类型 | 作用 | 示例 |
| --- | --- | --- |
| 内容标签 | 平台理解账号主题 | 护肤、职场、旅行、摄影 |
| 人群标签 | 平台理解目标用户 | 学生党、宝妈、打工人、新手 |
| 场景标签 | 平台理解使用时机 | 通勤、宿舍、约会、周末 |
| 风格标签 | 用户形成记忆点 | 清单型、测评型、故事型、吐槽型 |
| 商业标签 | 品牌判断合作适配度 | 美妆、食品、家居、数码 |

机器人应在每条笔记生成时检查：内容是否强化了至少 2 个账号核心标签。

### 2.3 冷启动目标

冷启动阶段建议追踪“信号质量”，而不是只追求单篇爆款。

| 阶段 | 目标 | 核心动作 | 观察指标 |
| --- | --- | --- | --- |
| 0-10 篇 | 确认定位是否可讲 | 测试 2-3 个内容支柱 | 点击率、完读感、收藏 |
| 10-30 篇 | 找到可重复栏目 | 固定栏目名、结构和封面风格 | 收藏率、关注转化 |
| 30-60 篇 | 放大有效选题 | 复刻高表现角度，减少低效内容 | 粉丝增长、搜索流量 |
| 60 篇后 | 准备商业化 | 建立账号数据卡、案例库、报价逻辑 | 商单询盘、私信质量 |

## 3. 内容生产系统

### 3.1 选题池

选题池要按“用户需求”组织，而不是按创作者灵感组织。

| 选题类型 | 适合目的 | 示例结构 |
| --- | --- | --- |
| 痛点解决 | 获得收藏和关注 | “新手做 X 最容易踩的 5 个坑” |
| 清单合集 | 获得收藏 | “预算 100 元以内的 X 清单” |
| 对比测评 | 建立信任 | “A 和 B 到底怎么选” |
| 经验复盘 | 建立人设 | “我踩过的 X 坑，重新做会这样选” |
| 趋势热点 | 扩大曝光 | “最近很火的 X，适合哪些人” |
| 商单软植入 | 测试转化 | “解决某场景问题时，我会用到的工具” |

机器人生成选题时，应输出：

- 选题标题
- 目标人群
- 用户痛点
- 内容价值
- 内容类型
- 适合账号阶段
- 预期指标
- 风险提示

### 3.2 标题钩子

标题的任务是制造点击动机，但不能用虚假信息骗点击。

常见标题公式：

| 公式 | 适用场景 | 示例 |
| --- | --- | --- |
| 痛点 + 解决 | 教程、攻略 | “总是选错 X？先看这 5 个判断标准” |
| 人群 + 场景 | 垂直人群 | “学生党第一次买 X，别先看颜值” |
| 数字 + 结果 | 清单、复盘 | “7 个让我效率翻倍的小习惯” |
| 对比 + 选择 | 测评、购买决策 | “A 和 B 都用过，最后留下的是这个” |
| 反常识 | 观点、经验 | “X 不是越贵越好，关键看这 3 点” |
| 避坑提醒 | 高风险决策 | “买 X 前先确认这件事，真的能省钱” |

不建议使用：

- 绝对化承诺：必爆、稳赚、100% 有效。
- 虚假身份：假装医生、律师、官方人员或品牌内部人士。
- 虚假体验：未使用却写成亲测。
- 恶意引战：刻意制造群体对立或诱导骂战。
- 欺骗性标题：标题承诺的内容正文没有兑现。

### 3.3 封面策略

封面是内容的第一层信息压缩。

封面应回答：

1. 这篇内容讲什么？
2. 适合谁看？
3. 看完能得到什么？
4. 为什么现在要点进来？

封面常用结构：

- 大标题：不超过 12-16 字。
- 副标题：补充人群、场景或收益。
- 视觉主体：产品、人物、场景、对比图、清单截图。
- 风格统一：同一栏目尽量保持字体、色彩和版式一致。

机器人应生成：

- 封面主标题
- 封面副标题
- 画面主体
- 色彩/风格建议
- 是否适合做系列模板

### 3.4 正文结构

正文的任务是兑现标题承诺。

推荐结构：

```text
开头：说明痛点或场景，让用户确认“这和我有关”
中段：给出清单、步骤、对比、经验或案例
结尾：总结结论，给出收藏/评论/下一步行动理由
```

正文常用框架：

| 框架 | 适合内容 | 结构 |
| --- | --- | --- |
| 清单型 | 收藏向 | 背景说明 → 清单条目 → 使用建议 |
| 避坑型 | 决策向 | 常见错误 → 后果 → 正确做法 |
| 测评型 | 种草/接广告 | 使用场景 → 体验维度 → 适合/不适合人群 |
| 故事型 | 人设向 | 真实经历 → 转折 → 方法 → 结果 |
| 对比型 | 购买决策 | A/B 差异 → 适合人群 → 选择建议 |
| 教程型 | 搜索向 | 前置条件 → 步骤 → 注意事项 |

### 3.5 标签策略

标签不是越多越好，而是帮助平台理解内容。

推荐标签组合：

- 2-3 个领域词：如 `#护肤`、`#职场效率`。
- 1-2 个人群词：如 `#学生党`、`#打工人`。
- 1-2 个场景词：如 `#通勤好物`、`#租房改造`。
- 1-2 个内容形式词：如 `#避坑指南`、`#好物分享`。
- 0-1 个热点词：只在真实相关时使用。

### 3.6 评论区引导

评论区引导应促进真实交流，而不是制造虚假互动。

合规引导：

- “你更关心 A 还是 B？”
- “还有哪些场景想看？”
- “如果你也遇到过，可以补充经验。”
- “需要清单版可以留言，我后续整理。”

不合规或高风险引导：

- 用小号伪造争议。
- 诱导刷屏、互赞、互藏。
- 用虚假抽奖诱导互动。
- 使用“评论区吵起来”等恶意引战策略。

## 4. 养号运营

### 4.1 养号的正确含义

本文档中的“养号”指：通过稳定、真实、垂直、持续的内容输出，让账号逐步获得平台识别、用户信任和商业合作能力。

不包含：

- 批量注册账号。
- 脚本登录、脚本浏览、脚本点赞。
- 高频机器发布。
- 伪造评论、收藏、转发。
- 绕过平台风控。

### 4.2 内容稳定性

账号应保持稳定的内容支柱。

示例：

| 内容支柱 | 目的 | 比例 |
| --- | --- | --- |
| 教程/攻略 | 搜索流量和收藏 | 40% |
| 测评/对比 | 信任和商业化 | 25% |
| 经验/故事 | 人设和关注 | 20% |
| 热点/趋势 | 曝光和破圈 | 10% |
| 商业合作 | 变现 | 5% |

机器人应按周检查栏目比例，避免连续多篇内容偏离定位。

### 4.3 发布节奏

发布节奏要稳定，但不应追求机器高频。

建议：

- 新号：每周 3-5 篇高质量内容，优先测试定位和栏目。
- 稳定期：每周 2-4 篇，保持固定栏目。
- 商业化期：自然内容和商单内容穿插，避免连续多篇硬广。

机器人只维护内容日历和提醒，不自动代发。

### 4.4 爆款复盘

爆款不是只看点赞数，而要拆解产生原因。

复盘维度：

| 维度 | 问题 |
| --- | --- |
| 选题 | 是否击中了高频痛点或强需求？ |
| 标题 | 是否清楚承诺了收益？ |
| 封面 | 是否一眼看懂？ |
| 正文 | 是否兑现标题？ |
| 标签 | 是否匹配人群和场景？ |
| 评论 | 是否出现真实问题和补充经验？ |
| 转化 | 是否带来关注、私信或商单询盘？ |

复盘结论必须落到下一步动作：

- 复刻该选题。
- 改写成系列。
- 做对比/清单/教程变体。
- 调整封面风格。
- 放弃低效角度。

### 4.5 低表现内容修正

低表现内容不一定是坏内容，可能是入口表达失败。

常见问题与修正：

| 问题 | 表现 | 修正 |
| --- | --- | --- |
| 标题太泛 | 点击低 | 改成人群 + 痛点 + 收益 |
| 封面不清楚 | 点击低 | 增加大字标题和视觉主体 |
| 正文太散 | 收藏低 | 改成清单或步骤 |
| 商业味太重 | 互动低 | 增加真实场景和适合/不适合人群 |
| 定位偏离 | 关注低 | 回到账号核心内容支柱 |

## 5. 接广告能力

### 5.1 接广告前的账号资产

账号具备接广告能力，不只看粉丝数，还看内容质量、粉丝匹配度和商业表达能力。

基础资产：

- 账号定位说明。
- 粉丝画像。
- 近 30/60/90 天数据。
- 高表现笔记案例。
- 可合作品类。
- 不接品类。
- 报价区间。
- 交付形式。
- 过往合作复盘。

### 5.2 商单筛选

接广告前必须判断品牌是否适合账号。

筛选字段：

| 字段 | 判断 |
| --- | --- |
| 品类适配 | 是否符合账号领域和粉丝期待 |
| 产品可信 | 是否能真实体验、真实表达 |
| 预算匹配 | 是否覆盖创作成本和账号价值 |
| 内容自由度 | 是否允许按账号调性创作 |
| 合规风险 | 是否涉及夸大功效、医疗疗效、金融收益等 |
| 交付要求 | 是否存在不合理修改、虚假承诺、刷量要求 |

建议拒绝：

- 要求虚假体验。
- 要求保结果、保转化。
- 要求不标注商业合作但明显是广告。
- 要求删除真实缺点。
- 要求导流到高风险渠道。

### 5.3 Brief 拆解

品牌 brief 应拆成可执行信息。

```text
品牌/产品：
合作目标：
目标人群：
核心卖点：
禁用表达：
必须露出：
内容形式：
发布时间：
审核节点：
交付物：
数据回传：
```

机器人应根据 brief 输出：

- 内容角度建议。
- 标题候选。
- 卖点转译。
- 适合/不适合人群。
- 风险表达替换。
- 交付清单。

### 5.4 报价逻辑

报价不是只按粉丝数计算，应综合内容价值和交付成本。

影响因素：

- 账号垂直度。
- 粉丝质量。
- 近 30 天平均阅读。
- 收藏/评论/关注转化表现。
- 内容制作复杂度。
- 是否需要拍摄、出镜、剪辑。
- 授权周期。
- 排他要求。
- 修改轮次。
- 是否包含二次分发授权。

报价单应包含：

- 单篇图文报价。
- 单篇视频报价。
- 套餐报价。
- 授权费用。
- 修改轮次。
- 交付周期。
- 付款方式。
- 取消/延期规则。

### 5.5 商单复盘

商单复盘不仅给品牌看，也用于优化账号商业化能力。

复盘报告结构：

```text
合作目标：
内容链接：
发布时间：
曝光/阅读：
点赞：
收藏：
评论：
关注增长：
私信/询盘：
评论区反馈摘要：
高表现原因：
可优化点：
后续合作建议：
```

## 6. 自动化系统蓝图

### 6.1 总体原则

机器人定位为“合规内容运营助手”。

允许：

- 生成选题。
- 生成文案草稿。
- 生成封面建议。
- 生成发布计划。
- 整理复盘结论。
- 管理商单资料。
- 标记风险表达。

不允许：

- 自动注册账号。
- 自动发布笔记。
- 自动点赞、收藏、评论、关注。
- 模拟真人浏览。
- 制造虚假互动。
- 绕过平台限制。
- 生成虚假体验和夸大承诺。

### 6.2 模块划分

| 模块 | 输入 | 输出 | 约束 |
| --- | --- | --- | --- |
| 账号画像模块 | 领域、人群、目标、边界 | 账号定位、内容支柱、风格规则 | 不生成违规定位 |
| 选题生成模块 | 账号画像、热点、痛点库 | 选题池、优先级、内容类型 | 不追逐无关热点 |
| 笔记生成模块 | 选题卡、内容素材、风格 | 标题、封面、正文、标签、CTA | 不虚假体验、不夸大 |
| 发布计划模块 | 内容池、栏目比例、节奏 | 内容日历、任务状态 | 不自动代发 |
| 复盘模块 | 笔记数据、评论反馈 | 复盘结论、下次动作 | 不做刷量建议 |
| 商单模块 | 品牌 brief、报价规则 | 合作记录、交付清单、复盘报告 | 不承诺保效果 |
| 合规模块 | 文案、行为、brief | 风险等级、修改建议 | 拦截高风险内容 |

### 6.3 核心工作流

```text
账号画像 → 选题池 → 笔记 brief → 文案/封面草稿 → 合规检查 → 人工确认 → 发布记录 → 数据复盘 → 策略更新
```

商单工作流：

```text
品牌咨询 → 品类适配检查 → brief 拆解 → 报价确认 → 内容草稿 → 合规检查 → 人工发布 → 数据回收 → 复盘报告
```

### 6.4 人工确认点

必须人工确认：

- 账号定位最终版。
- 是否接某个品牌。
- 商业报价。
- 发布前最终文案。
- 是否含商业合作标注。
- 是否真实体验过产品。
- 是否提交品牌审核。
- 是否发布和删除内容。

## 8. 评估指标

### 8.1 内容指标

| 指标 | 含义 | 用途 |
| --- | --- | --- |
| 曝光 | 内容被推荐或展示的次数 | 判断平台分发 |
| 阅读/观看 | 用户进入内容的次数 | 判断入口吸引力 |
| 点赞 | 轻互动 | 判断情绪认同 |
| 收藏 | 强价值信号 | 判断实用价值 |
| 评论 | 交流信号 | 判断话题空间 |
| 关注 | 账号转化 | 判断内容与账号定位匹配 |
| 私信 | 商业或深度需求 | 判断转化潜力 |

### 8.2 机器人质量指标

| 指标 | 说明 |
| --- | --- |
| 选题命中率 | 生成选题中被采用的比例 |
| 标题采用率 | 标题候选被人工选中的比例 |
| 合规拦截率 | 成功识别高风险表达的比例 |
| 复盘可执行率 | 复盘结论能转化为下一步动作的比例 |
| 商单适配率 | 推荐合作中被确认适配的比例 |
| 人工修改量 | 草稿到发布稿的修改幅度 |

## 9. 合规规则库

### 9.1 内容质量规则

- 标题承诺必须在正文兑现。
- 正文必须包含具体信息，不能只有情绪。
- 测评类内容必须说明使用场景和适合/不适合人群。
- 商单内容必须避免伪装成无利益关系的纯自发推荐。
- 医疗、金融、教育、功效类内容需要额外谨慎，避免绝对化结论。

### 9.2 垂直度规则

- 每条笔记必须至少关联一个核心领域标签。
- 每周内容应覆盖主要内容支柱，不应连续偏离定位。
- 热点内容必须能回到账号领域，否则不建议追。
- 商单品类必须与账号定位和粉丝期待匹配。

### 9.3 发布节奏规则

- 机器人只生成内容日历和提醒，不自动发布。
- 不建议机器高频发布。
- 商单内容之间应穿插自然内容，避免账号商业味过重。
- 低质量内容不应为了频率强行发布。

### 9.4 商单合规规则

- 不接要求虚假体验的合作。
- 不接要求刷量、买互动、伪造数据的合作。
- 不承诺保曝光、保销量、保转化。
- 不使用未经验证的功效表达。
- 不隐藏必要的商业合作信息。
- 不接受超出实际使用体验的品牌指定话术。

### 9.5 风险表达替换

| 高风险表达 | 替换建议 |
| --- | --- |
| “必爆” | “更容易获得点击/互动” |
| “100% 有效” | “在某些场景下可能有效” |
| “用了立刻见效” | “个人体验中感受到变化” |
| “全网最低” | “价格有优势，需以实际页面为准” |
| “闭眼买” | “适合这些人优先考虑” |
| “不买不是人” | “如果符合需求，可以加入备选” |
| “评论区吵起来” | “欢迎分享不同使用体验” |

## 14. 实施优先级

如果后续开发机器人，推荐按以下顺序实现：

1. **账号画像与规则库**：先建立边界，防止机器人生成偏离定位或高风险内容。
2. **选题生成与内容日历**：解决持续输出问题。
3. **笔记生成与合规检查**：生成可编辑草稿，而不是直接发布内容。
4. **复盘记录与策略更新**：形成数据闭环。
5. **商单管理模块**：在账号有基础数据后承接商业化。

最小可用版本只需要：

- `account_profile`
- `topic_card`
- `note_brief`
- 合规检查规则
- 人工确认流程

不要在第一版实现自动发布、自动互动或任何平台行为模拟能力。
