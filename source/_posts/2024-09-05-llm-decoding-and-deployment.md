## 解码与部署

### 解码策略
LLM 本质上是进行概率采样的过程，需要合适的解码策略让其输出合理的内容。下面介绍 LLM 常见的解码策略及相关的超参数。

我们主要讨论自回归场景下的解码策略。无论模型采用了什么样的 Coder 模式，自回归的整体思路都是：让模型根据上下文词元序列建模下一个词的概率分布，按一定策略选择下一个词，重复这个步骤直到输出结束。

一个直观的解码策略是贪心策略（Greedy）：在每一步上都选择概率最高的词元。贪心策略是确定性策略，执着于局部最优，因此在一些开放式生成任务中（如文本创作与故事生成），贪心策略可能因此生成不自然、重复的句子。

与贪心相对的是概率采样（Probability Sampling），根据模型建模的概率分布采样下一个词元，增强生成过程的随机性。

这两种经典的采样手法都有对应的改进形式。对于贪心策略，有：

* 束搜索：保留前 $n$ 个概率最高的句子，最终选择整体概率最高的回复。这里的 $n$ 称为束的大小，当其为 1 时退化为贪心搜索。束搜索会生成多个回复，最终选择最合适的那一个。
* 长度惩罚：当缺少长度惩罚时，束搜索可能偏向更短的句子，因此需要在生成概率上增加惩罚系数 $\alpha$。
* 重复惩罚：为避免贪心搜索生成连续的重复词元，我们也会增加惩罚项。这种惩罚同样适用于概率随机采样。

概率随机采样在整个词表中选择词元，因此可能产生不相关的词元。为了提高生成质量，需要针对性地改进，避免其影响最终的生成结果。

* 温度采样：温度调节 Softmax 中的温度系数。降低温度系数会让概率分布更加集中，增加高概率词元被采样的可能性；当温度接近 0 时退化为贪心搜索，接近无穷时则趋近于随机采样。
* Top-k 采样：与温度不同，Top-k 采样直接剔除低概率词元——选择概率最高的前 k 个词元进行采样。
* Top-p 采样：Top-k 中固定的常数 k 可能无法适应不同的上下文环境。Top-p 采样要求采样的词元累积概率之和大于预设值，从而综合处理确定性与非确定性的生成任务，避免过大的 k 引入错误答案。

**现在大部分可调用的 API 都集成了多种解码方式，可以根据其文档来了解如何进行设置。理解解码相关参数的含义，就是本节的意义所在。**

### 低资源部署策略
LLM 参数量巨大，解码阶段需要占用大量显存资源，部署代价非常高。这里介绍最常用的模型压缩方法——模型量化（Model Quantization），用于减少 LLM 的显存占用。

在神经网络压缩中，量化通常表示从浮点映射到整数的过程。目前比较常用的是 8bit 量化，即 INT8 量化。针对神经网络模型，通常需要量化模型权重参数与激活值这两类数值。

量化的过程可以表示为将连续输入映射到离散输出集合的函数，这个过程涉及四舍五入与截断近似。量化也需要选取参数，但一般由模型研发人员提前准备好，无需使用者考虑。

在 LLM 领域，量化方法基本分为两种：量化感知训练（QAT）与训练后量化（PTQ）。前者需要更新参数权重完成模型量化，后者则无需，因此 PTQ 使用得更为广泛。在实际应用中，QLoRA 作为一种兼容微调与量化步骤的方法，可以同时结合 LoRA 的轻量化优点与量化的模型压缩能力，以 4bit 精度保持原始模型的能力。

根据现有的 LLM 研究，可以给出下面的经验：

* INT8 权重量化对模型性能的影响很小，更低精度的量化效果则取决于具体方法。
* 对于 LLM，激活值相对于模型权重更加难以量化。
* 轻量化微调方法可以补偿量化带来的损失。LoRA 与 QLoRA 技术被广泛用于量化领域，具有良好的应用前景。

### 其他模型压缩方法
模型蒸馏与剪枝是另外两种常见的模型压缩方法。它们都旨在精简模型的复杂结构，从而减少所需的参数数量，这个思路与量化完全不同。

目前 LLM 领域的知识蒸馏方法都专注于黑盒蒸馏研究，即利用教师模型的输出信息来训练更小的学生模型。典型工作主要关注上下文学习能力、思维链能力等。

LLM 的剪枝则主要在学术界进行研究，借鉴传统神经网络与树模型的剪枝方法来处理 LLM，一般可以减少约 50% 的参数量。
