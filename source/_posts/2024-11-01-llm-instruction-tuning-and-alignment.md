## 指令微调
指令微调（Instructions Tuning）指的是使用自然语言形式的数据对预训练后的 LLM 进行参数微调，也可以称为监督微调（Supervised Fine-Tuning，SFT）。指令微调需要先构造指令化的实例，然后通过有监督的方式对 LLM 的参数进行微调，从而让模型可以处理各种下游任务而不需要额外的学习。

指令微调是一种通用的模型能力增强方法，只需少量数据就能提升任务求解能力并完成领域专业化适配。
### 指令数据构建
一个经过指令格式化的数据实例包括任务描述（指令）、任务输入、任务输出以及可选的示例。下面介绍三种构建格式化指令数据的方法。

#### 基于现有的NLP任务数据集构建
学术界围绕 NLP 任务积累了大量的数据集，这些都是重要的监督学习资源，可以用于指令数据集的构造。这类任务都包含输入和输出两个部分，如翻译任务有两种语言的对照，概括任务有长文本与短文本。为了构造指令化训练数据，只需要为上述输入与输出增加任务描述信息，就可以把一个数据实例全部使用自然语言进行表达。

如：请回答下面问题，Answer，Question；请翻译中文为英文，中文部分，英文部分。

经过指令数据集微调后，大语言模型可以学习到遵循指令的能力，从而解决其他从未见过的 NLP 任务。研究已经证明任务描述对于指令微调非常重要。也可以通过将问题-答案对颠倒来得到新的实例。

#### 基于日常对话数据构建
仅基于 NLP 实例的多样性受限于学科任务，与人类真实需求也无法完全匹配，因此研究者开始使用日常对话的实际需求作为任务描述。InstructGPT 使用用户提交给 OpenAI 的查询作为任务描述，采取人工标注的多轮对话训练，如：

第一轮输入：激活函数有什么作用

第一轮输出：xxxxx 非线性性

第二轮输入：什么是复杂的非线性关系

第二轮输出：xxxxxxx

#### 基于合成数据
为了减轻人工收集与标注数据的负担，研究者提出了半自动化的数据合成方式：借助已经过良好训练的 LLM，将现有指令数据输入给模型，要求其生成大量的任务描述与输入、输出数据。

Self-Instruct 使用 100 多个人工撰写的示例作为初始任务池，通过提示大语言模型的方式生成新的指令微调数据。也有团队以现有文本作为输出，利用上下文学习逆向合成对应的输入与任务指令，确保模型输出准确而流畅的文本。

合成数据的核心在于撰写提示词，让 LLM 根据现有要求与示例生成微调指令；后续对生成出的指令进行下一阶段处理、保证指令质量同样重要。

#### 指令数据构建的提升方法
指令数据的格式与数量对后续性能都会产生影响，下面介绍如何构建高质量的指令数据集。

指令格式对性能影响很大，现有研究表明在输入中附带适量实例可以提升实际性能。为了激发模型逐步推理的能力，研究者尝试在数据中增加思维链样本，也起到了不错的效果。

指令数量对模型性能的影响学界也有讨论。现有研究表明，对于已经过充分预训练的基座模型，只需一定数量的高质量指令就可以激活大语言模型的能力——几万甚至几千条指令就已经足够。

也有研究考虑对现有指令数据集进行重写与筛选，以提高多样性与质量，这本质上也是一种合成指令。

**指令的质量比数量更重要，采用闭源 LLM 合成与重写现有指令可以弥补当前数量上的不足**


### SFT训练策略
在训练方式上，指令微调与预训练相似，数据组织形式可以参考预训练阶段采用的技术。这里介绍一些特有的训练策略。

目标函数调整：在 SFT 阶段，只需计算输出部分的损失，因为这本质上是一个监督学习任务。

Batchsize 与学习率：SFT 通常只需采用较小的 Batchsize 与学习率，Batchsize 在 10-100 之间，学习率控制在 $10^{-5}$ 至 $10^{-6}$。

多轮对话数据：可以拆分成多个对话单独训练，通常采用掩码方式实现一次性输入但分别对每一轮对话计算损失。

单一指令的数据只能增强单方面能力，如果需要增强全面的能力，应当混合各类数据实现大模型指令微调。由于各类数据之间差异较大，可以采取多阶段微调，逐步提高指令的复杂程度。

### 高效的SFT
全参数微调仍需较大的算力资源，本节考虑如何进行轻量化的参数微调，用更少的算力实现近似的效果。使用最广泛的方法是低秩适配（Low Rank Adaptation，LoRA）微调技术。

#### LoRA基础
大语言模型中包含大量的线性变换层，其中参数矩阵的维数通常非常高。针对特定任务进行适配时，参数更新通常存在一个较低的内在秩。为了利用这一性质，LoRA 提出在参数矩阵上增加一个低秩分解矩阵来近似每层的参数更新，减少下游任务所需要训练的参数量。

基本的参数矩阵 $W$ 的更新表达为
$$W=W_{0}+\Delta W$$
LoRA 的基本思想是冻结原始矩阵 $W$，通过低秩分解矩阵近似 $\Delta W = AB^T$。
在整个微调期间，原始参数矩阵 $W_0$ 不会更新，分解矩阵 $AB$ 是可训练参数，用于适配下游任务，前向传播过程变为
$$h=W_{0}x+AB^{T}x$$
微调完成后，参数矩阵会进行合并，解码过程的开销不会发生变化。LoRA 技术可以有效降低 SFT 所需要的显存数量。

**由于LoRA依赖附加参数实现，具有可插拔性：训练好的 LoRA 可以被共享，多个任务的不同 LoRA 可以被组合，因此 LoRA 有很好的应用前景**
#### LoRA变种
一个基础的 LoRA 变种 AdaLoRA 讨论动态调整秩的大小，让更重要的参数矩阵拥有更高的秩，实现动态的资源节省。

QLoRA 技术将原始参数矩阵量化为 4bit，保留 LoRA 部分的参数为 16bit，在保持微调性能的同时进一步节省了显存开销，实现了在一张 48G 显存的显卡上微调 65B 的 LLaMA 模型。

LoRA 还有包括 DyLoRA、DoRA 在内的多种变种，在需要的时候另行介绍。有些不实用的变种可能未被包含在基础的微调工具中。

为了组合多种 LoRA，可以参考 LoRAhub 技术，通过组合和适应两个阶段将来源不同的 LoRA 组合为一个整体，实现在新任务上的泛化。
#### LoRA应用
LoRA 技术并非发源于 LLM 领域，而是图像处理领域，但 LLM 的兴起促使 LoRA 这种高效微调方法得到了广泛分析。在 100 个自然语言处理任务上全面对比全参数微调与 LoRA 后，发现它可以在保持模型性能的同时显著降低训练成本，因此得到了广泛应用。

（参数高效微调的深入讨论见 [PEFT：参数高效微调](/blog/2026/03/05/peft-parameter-efficient-fine-tuning/)；RL 对齐的深入讨论见 [RL Alignment：从 Reward 到 Advantage](/blog/2026/03/16/rl-alignment-from-reward-to-advantage/)）

### 参数附加与参数选择
除了最常用且经典的 LoRA 方法以外，还有两种 LLM 的高效微调方法值得介绍，它们包含了不同的微调思想，也可以与 LoRA 方法结合使用，分别是参数附加方法与参数选择方法。

前者冻结模型的原始参数，在模型中附加新的小型可训练模块，通过微调这些新加入的模块实现高效微调。新加入的模块一般称为适应层（Adapter Layer），被插入到模型不同层之间补充特定任务信息。

后者选择模型的一部分参数进行微调而冻结其余参数，其核心在于找到那些对下游任务至关重要的参数层，通过选择性微调在降低计算负担的同时提升模型性能。

#### 参数附加方法
参数附加方法通过增加和训练新的附加参数对 LLM 进行微调，可以将新参数增加在模型输入、模型输出以及模型本身。

加在输入的方法将额外参数附加到模型的输入嵌入中。最经典的方法是 Prompt-tuning，它在原本的嵌入矩阵 $X$ 上合并软提示矩阵 $R$，通过拼接得到更大的矩阵再输入给模型的 Transformer 结构，通过反向传播优化矩阵 $R$。

通过参数附加与微调，模型可以将学到的信息存储到软提示矩阵中，从而习得领域知识。软提示只需较短长度就能发挥作用，长度一般设置为几十到几百不等。

加在模型的方法也是向模型中添加软提示。与只在输入嵌入中添加不同，此时的附加参数会被添加到注意力模块中，在 QKV 中扩展长度来附加信息。也有的方法通过在全连接层之后增加信息适配层。

加在输出的方法完全不修改模型的权重，通过访问模型输出词汇表的预测分布来实现定制化调整，利用一个小型的反专家模型来辅助微调。

#### 参数选择方法
参数选择方法无需添加额外参数，避免了推理阶段引入额外计算成本。通过冻结部分参数、微调部分参数来实现信息引入。由于 LLM 目前仍为黑盒，选择微调哪些参数完全依赖于个人经验，缺少系统的理论支撑。

实验验证参数选择方法是有效的，但如何使其发挥出最好效果还没有被研究清楚。


## 人类对齐
大语言模型的能力源于海量文本数据的学习，因此其行为会受数据质量与数据来源等多方面因素的影响，可能产生偏见、冒犯与事实错误。如何让 LLM 在训练过程中保持与人类一致的价值观，称为人类对齐。这一章将系统讨论人类对齐问题。
### 人类对齐的背景与标准
人类对齐的核心背景在于输出符合人类价值观的内容，即便遭受了一定程度的误导也要保持正确的价值观进行回答。

对齐的三个基本标准为
1. 有用性：LLM 需要提供有用的信息，正确理解上下文
2. 诚实性：LLM 的输出应该保持真实与客观，不产生误导性陈述
3. 无害性：LLM 不应该输出可能引发负面影响与危害的内容

这些指标都基于人类认知进行设计，通过简单的优化目标难以直接建模，这就是后面要引入基于人类反馈的强化学习的原因。

### 基于人类反馈的强化学习
由于对齐标准难以通过形式化的优化目标建模，研究者提出了基于人类反馈的强化学习（Reinforcement Learning from Human Feedback，RLHF），引入人类反馈对 LLM 的行为进行指导。

RLHF 首先收集人类反馈数据训练奖励模型，然后根据奖励模型使用强化学习算法（如 PPO）对 LLM 进行微调。这已成为人类对齐的主要途径。

RLHF 主要包含三个部分：待对齐模型、奖励模型、强化学习算法；其中后两者是这里需要主要介绍的。

RLHF 最核心的部分是奖励模型训练。语言模型生成一系列候选输出，由标注员进行偏好标注（主要是排序），再使用这些数据训练奖励模型来预测排序。奖励模型的具体训练细节从略。

RLHF 的下一部分是强化学习训练。语言模型对齐被转换为一个强化学习问题：它接受输入并输出文本，动作空间是所有词元，状态是当前词元序列。奖励模型根据状态提供奖励分数，指导策略模型的优化。通常会增加一个正则项避免策略模型偏离原始模型过远。
### 非强化学习对齐
非强化学习方式希望通过高质量的对齐数据集，使用特定的监督学习算法对 LLM 进行微调。这类方法需要精心构造的高质量对齐数据集，其中蕴含人类价值观，以正确回应人类指令。

与传统的 SFT 不同，这种方法需要优化过程能够区分对齐数据与未对齐数据，从而快速从数据中学习到人类期望的行为模式。

目前构造数据的方式主要分为两种：基于奖励模型与基于 LLM 自身。
* 基于奖励模型的方法使用训练好的奖励模型评估对齐程度，根据不同分数得到各组与人类偏好对齐程度不同的数据，可用于后续监督微调，避免生成质量过低的回答
* 基于 LLM 的方法希望模型进行自我纠正，通过自我评价或多个 agent 之间的交互获取后续的人类对齐数据

DPO 算法是一种较主流的无需强化学习的对齐算法，其复杂度远低于传统强化学习算法，与直接进行监督微调的复杂度接近，并且更容易确定超参数。
### SFT与RLHF
关于 SFT，人们普遍认为它用于解锁 LLM 的能力，而不是增加新的能力。当尝试增加模型不具备的能力时，幻觉问题将非常明显。这意味着需要关注模型的能力边界，减少微调的负面效应。

作为一种模仿学习算法，SFT 旨在模仿标注数据的行为，而不是进行探索。而标注数据的风格、创作水平、主题偏好等经常存在一定差异，这种差异会影响 SFT 的学习性能。

RLHF 可以在减少有害内容输出的同时，增强模型的综合能力。它无需模仿，不被强迫学习示例数据，可以缓解 SFT 由于知识边界产生的幻觉问题。但它也继承了强化学习的全部缺点，训练起来非常困难。

相比之下，SFT 更适合训练结束后增强模型性能，实现简单、快速高效。模型对齐的研究还有很大的发展空间。
