## 规划与智能体

规划是指为任务目标制定包含一系列动作的解决方案。它体现了 LLM 处理复杂任务的能力，也是实现智能体的核心能力。智能体是 LLM 的重要应用方向之一，能够自主完成任务目标而无需额外帮助。本章围绕这两个主题展开讨论。

### 基于 LLM 的规划

上下文学习与思维链等提示方式简洁通用，但对复杂任务表现仍然较差。需要利用 LLM 进行规划——将复杂任务分解为若干关联的子任务。围绕子任务制定包含一系列执行动作的解决方案，可以简化任务难度。实现 LLM 的规划也是构建智能体（Agent）与多智能体系统（Multiple Agents）的技术基础。

基于 LLM 的规划方法包含三个组件：任务规划器（Task Planner）、规划执行器（Plan Executor）以及环境（Environment）。LLM 作为任务规划器，生成一系列动作作为目标任务的解决方案，并用合适的方式表达每个动作。规划执行器负责执行动作——根据动作的不同，执行器可以是 LLM，也可以是调用预定义函数或激活外部真实世界的实体。环境则是执行动作并给出反馈的地方。

规划器可以一次性生成全部动作，也可以根据执行反馈修改和优化下一步方案，迭代完成任务。

无论何种规划，都基于提示词来让模型生成。MCP 方案就是一种典型的规划——让模型根据现有提示选择合适的工具，执行对应工具后再让其继续处理。为了利用上下文学习能力，最好在提示词中提供一些示例。

### 基于 LLM 的智能体

智能体（Agent）是一个具备环境感知、决策制定与动作执行的自主算法系统。研发智能体的初衷是模拟人类与其他生物的智能行为，自动解决任务。传统智能体技术受预设规则限制，能力不足。LLM 在处理复杂任务上展现了优秀的能力，用 LLM 作为智能体的核心组件，就形成了基于 LLM 的智能体。

基于大语言模型的智能体围绕三个基本组件构造：记忆组件（Memory）、规划组件（Planning）和执行组件（Execution）。通过组件的协作，智能体得以感知环境、决策并执行，最终完成相应任务。

LLM 智能体通过特殊设计的读写操作，将相关信息存储在短期与长期记忆中。面临不同需求时，它可以灵活调用记忆来支持认知与推理。短期信息通常对应模型的上下文窗口，只会使用一次。必要时，短期记忆会被转为长期存储，以文本或结构化数据库的形式保存在外部。LLM 通过检索机制读取长期记忆。

规划组件类似于前面讨论的任务规划器。它引入类似人类的思考方式，将复杂任务分解并逐步处理，得到最优方案后执行。

执行组件也类似于前面讨论的执行器。通过执行组件，Agent 与环境交互并获得实际反馈。执行组件可以基于语言模型完成任务，也可以集成外部工具来增强能力。

顺序执行这三个组件，智能体就能处理用户提出的复杂任务。

### LLM 多智能体系统

与单一智能体的独立工作模式不同，多智能体强调智能体之间的协作，发挥集体优势。在 Multiple Agents 研究中，我们会实例化多个智能体，各自扮演特定角色并承担对应功能。通过交互协作处理更复杂的任务。要确保智能体之间的协作，交互方式至关重要。

通讯与协同是智能体协作的基础，用来实现信息共享、任务分配与协同控制。通讯机制包含三个基本要素：协议、拓扑、内容——分别规定如何交换与共享信息、哪些智能体参与交换、交互哪些内容。协同机制包括协作、竞争、协商，决定了各智能体交换信息的目的。

### 智能体应用与遇到的问题

下面介绍三类典型智能体应用：

* **WebGPT**：一种单智能体，可以自主从互联网检索数据，并结合数据与用户进行自然语言交互。
* **MetaGPT**：一种多智能体系统，模仿人类组织的运作方式，模拟软件开发的角色与协作，实现软件开发工作。
* **西部世界沙盒模拟**：一种多智能体的社会模拟，各个角色根据人物背景在小镇中生活。

智能体系统当前面临的问题：

* 计算资源消耗巨大，单智能体和多智能体都面临这一挑战。
* 智能体需要调用外部工具实现特定功能。这种工具的开发需要与模型紧密结合，但相关工具链尚不完善。
* 多智能体系统中，随着智能体数量增加，协调和交互变得非常繁琐。有效协同工作需要更高效的通信与协调机制。
* 模型基础能力不足，需要针对智能体设计进行模型优化。

## 应用进展

语言模型是自然语言处理的重要研究方向。LLM 的出现助推了现有 NLP 系统的应用改进。零样本下的 LLM 性能喜人，但成本较高。

LLM 对传统信息检索与应用产生了重要影响。LLM 的语言理解、推理与生成能力可以帮助构建更有效的信息检索系统。目前主要方向包括直接基于提示词进行检索，以及 RAG（检索增强生成）技术。前者用于信息检索领域，后者则为 LLM 提供外部信息来增强生成内容的质量。

LLM 在推荐领域也有应用，包括直接基于 Agent 与智能体系统进行推荐，以及基于 SFT（监督微调）的实现。推荐领域的整体应用范围目前有限。

MLLM（多模态大语言模型）已经成为 LLM 领域的主要研究方向。处理更多数据类型对模型的广泛应用有着重要意义。

LLM 也可以与知识图谱进行交互，获得更多有助于高质量回答的上下文信息，辅助上下文学习。为了更好地获取知识图谱的信息，编写模型可以直接调用的函数非常重要。

**让模型拥有"手脚"，从广泛的外部获取信息辅助上下文学习，对现有模型非常重要。**
