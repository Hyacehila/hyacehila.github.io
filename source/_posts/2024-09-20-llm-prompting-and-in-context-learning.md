## 提示学习

在经历了预训练、微调与人类对齐之后，我们来讨论如何用合适的提示词让 LLM 解决实际任务，即通过自然语言接口与 LLM 交互。

### 基础提示

微调的代价仍然比较高，因此通过自然语言提示来激活 LLM 能力、解决下游任务成为了主要途径。提示的质量会很大程度上影响 LLM 在特定任务上的表现，本节展开对基础提示编写的介绍。

先来研究人工设计提示的要素与原则，它们将作为后续 Prompt Engineering 的基础。

提示词工程（Prompt Engineering）的核心是通过提示词将新任务构建为模型在预训练与微调阶段就已经熟悉的形式，利用模型泛化能力完成新任务。

针对 LLM 的提示设计，需要考虑下面四个关键要素：

1. 任务描述：向 LLM 展示应遵循的指令。需要清晰且具体地表达任务目标，对输入与输出的格式进行详细说明，使用关键词与特殊符号来强调特殊设置。
2. 输入数据：自然语言可用于描述输入数据的内容。特殊形式的格式化数据一般需要线性化为文本序列，也可以使用代码中的数据结构表示结构化数据（这类数据以外部程序直接读取居多）。
3. 上下文信息：对特定任务非常重要。将参考文档等内容以上下文形式引入提示词，可以结合这些信息来处理问题。后面会介绍的上下文学习也属于上下文信息的范畴（给出任务与解决的示例）。
4. 提示策略：有针对性的提示策略可以激活模型解决特定任务的能力，如「让我们一步一步思考」激活推理能力、「你是 xx 领域专家」激活特定领域能力。对于强化了对话能力的模型，可以将提示词拆分为多轮对话，逐步输入给 LLM。

根据上面的关键要素，可以给出一些经验原则：

1. 清晰表达任务目标（需要多轮调整来确认模型不会产生错误理解）。
2. 分解为简单而详细的子任务（将子任务显式地列出，要求模型逐步处理）。
3. 提供少量样本示例（参考上下文学习的思路，无需调整参数即可学习）。
4. 采用模型友好的提示格式：将任务描述、上下文信息、问题用符号分割。一般用 `###` 表示一个部分，用 `#` 表示一个子项，用 `\n` 换行分割语义。
5. 规范化输出格式（方便后续提取）。

如下例子：

```python
### 判断下面句子的情感为积极还是消极，并采用{"结果":""} 的JSON格式返回最终的结果

### 待分类的句子：xxxxxxxx
### 针对此问题有以下示例：
# xxx
# xxx
# xxx
```

关于提示词优化，可以基于 LLM 来进行。优化提示词本身也是一个可以交给 LLM 处理的任务：测试不同提示词在 LLM 上的效果，让 LLM 针对产生的问题进行优化，不断迭代。

（关于提示词优化的深入研究，参见 [Prompt Optimization: From Manual to Evolution](/blog/2026/04/26/prompt-optimization-from-manual-to-evolution/)）

### 上下文学习

上下文学习已成为 LLM 处理下游任务的主流途径。它使用任务描述和示例组成的自然语言文本作为提示，将示例按特定顺序组成提示内容，最后将测试样本放在末尾让模型产生输出。基于任务描述与示例信息，LLM 无需显式的参数更新就可以识别和执行新任务。

引入示例是上下文学习的重要特点，因此设计优秀的示例至关重要。示例需要包含丰富的任务信息，并与测试样本保持高度相关性。示例包含三个基本部分：

* 示例的输入
* 示例的输出
* 示例的指令

上下文学习（In-Context Learning）引入了 shot 的概念。不给出任何上下文、只有任务描述时，称为 zero-shot；给出一个示例时，称为 one-shot；给出多个示例时，称为 few-shot。

示例会明显地影响模型的效果，需要兼顾示例的**代表性与多样性**：前者决定单一任务上的性能，后者决定能否覆盖更多类型的任务。

RAG 技术可以看作一种更复杂的上下文学习，涉及更多关于示例检索的技术。其目的在于利用模型的上下文学习能力，通过特定语料库增强回答性能，减少模型幻觉。

上下文学习能力与模型自身的性能密切相关。如果需要更强的上下文学习能力，通常需要采用规模更大、在各测试榜单上排名更靠前的模型。

### 思维链提示

#### CoT 的基础介绍

思维链（Chain of Thought, CoT）是一种高级的提示策略，旨在增强 LLM 在推理任务上的表现。与上下文学习只采用「输入-输出」的二元对不同，思维链提示融合了推理步骤来指导从输入到输出的过程。

思维链是上下文学习的一种扩展形式，将「输入-输出」关系转换为「输入-思维链-输出」关系。思维链提供了一系列语义连贯且具有逻辑性的关键推理步骤，建立了输入与输出之间的连接，并通过激活推理能力来提升性能。

思维链的获取一般比较复杂，需要借助人工标注。DeepSeek-R1 采用的 Long-CoT 借用了思维链思想，但本质上使用了大量基于规则的 RL，属于微调范畴；而一般的 CoT 属于提示工程的一部分。

最基础的思维链提示是 `Let's think step by step`。这种指令可以在不额外提供思维链的情况下，让 LLM 先生成思维链再回答问题，从而提高准确率。

#### CoT 的几种扩展方法

有多种方法可以激活思维链。根据推理方式的不同，基于 CoT 的核心思想可以概括为三种：

* 按部就班：模型一步一步向后推理，形成逻辑连贯的链条。CoT、Zero-shot CoT、Auto-CoT 都属于这一类技术。
* 三思后行：模型每一步都停下来评估当前状况，从多个可能的方向中选择下一步的行进方向，在未知中探索最优的推理路径，以 ToT、GoT 为代表。
* 集思广益：模型同时生成多个推理路径并得到多个结果，最后汇总得到一个更全面而准确的答案，以 Self-Consistency 为代表。

#### CoT 原理

主流学者认为，思维链可以看作一种增强的上下文学习。当任务过于复杂时，基础提示无法充分表达任务意图，需要思维链来增强对意图的表示。也有研究发现，只要生成的文本包含显式的推理过程，就能改善模型的推理能力。思维链本质上是通过激发模型生成中间推理步骤，来提升其生成正确答案的概率。也就是说：**重要的不是思维链本身，而是模型的推理能力。**

CoT 的使用类似于上下文学习中的 shot 概念。只有问题、不给出上下文示例时，统称为 Zero-shot——对应的 Zero-shot CoT 只需添加一句 `Let's think step by step`。在上下文中给出同类型问题的示例时，则称为 few-shot，对应的版本称为 few-shot CoT。

### 角色扮演提示与情景提示

积极的心理暗示有助于激活模型的能力。这种心理暗示可以通过两种方式传递给 LLM：**角色扮演与情景代入。**

通过提示词指导 LLM 扮演特定角色来改善其相关技能，称为 Role-Playing 提示。设定一个详细的角色可以引导模型输出朝向所需方向，提供更高质量的回答。角色提示需要包含具体属性、职责、知识与技能。例如：「您是一名宠物营养顾问，负责为宠物设计符合其自然饮食习惯的健康营养方案。您精通 xx 宠物的饮食偏好与动物营养学知识。」+ 后续提示词。

情景代入是将特定情景下的专业知识与历史背景等信息加入到提示词中，让模型理解特定情景的环境，从而生成更丰富且符合情境的回答。
